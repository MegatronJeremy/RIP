Dataflow Engines shared over Infiniband
MPC-X series dataflow nodes provide multiple dataflow engines as shared 
resources on the network, allowing them to be used by applications running 
anywhere on the cluster . The MPC-X series provides maximum flexibility and 
performance density for the widest range of HPC applications. 
Tightly integrated Dataflow Engines and CPUs
MPC-C series nodes have the closest coupling of dataflow engines and multi-core 
CPUs within the same chassis. The MPC-C series are suitable for small systems or 
HPC applications requiring the most data transfer between CPUs and DFEs.
Low latency connectivity
MPC-N series nodes provide direct network connections to dataflow engines 
allowing ultra low latency line-rate processing of multiple 10-40Gbit data streams. 
Dataflow computing on the desktop
MaxWorkstation and MaxWorkstation10G are small form factor PCs that bring the 
power of dataflow computing to the desktop. Ideal for software developers as well 
as users who desire a dedicated compute resource on their desk.

HPC clusters with dataflow computing technology can be built around different combinations of 
MPC-X series and MPC-C series dataflow nodes.
•	MPC-X series systems provide both high density and flexibility. MPC-X nodes can be connected directly to CPU-only 
nodes via single or dual Infiniband cables, or via an Infiniband switch to share dataflow engines across the network.
•	MPC-C series systems are ideal for small deployments, or for applications that require the most data transfer between 
DFEs and CPUs.
Maxeler MPC nodes and CPU only nodes can be configured to match a wide range of application requirements. 
•	MPC-X series nodes can be configured with the type and number of DFEs, amount of DFE RAM & interconnect. Page 7
•	MPC-C series nodes can be configured with the type and number of DFEs, amount of DFE RAM, CPU type and main 
memory size, in-node disk storage and interconnect. Page 8
•	CPU only nodes can be configured to select CPU type and main memory size, storage and interconnect.  Page 10
You can choose a network architecture depending on the system arrangement you have selected and how you wish 
to integrate into the rest of the data center. Typically MaxRack systems contain a low-speed Ethernet management 
network (used to remotely monitor and control nodes) and a high-speed data network utilizing Ethernet and/or 
Infiniband. Options include:
•	Single 1Gb Ethernet network for data and management. Requires at least 1 Ethernet switch.
•	1Gb Ethernet management network, Infiniband data network. Requires at least 1 Ethernet and 1 Infiniband switch.
•	1Gb Ethernet management network, 1GbE or 10GbE data network. Requires at least 2 Ethernet switches.
•	1Gb Ethernet management network, Infiniband data network and Ethernet data network. Requires at least 2 Ethernet 
switches and 1 Infiniband switch.
“Cluster Management” nodes are standard CPU nodes which run software to perform functions such as job 
distribution, database hosting or system provisioning. If a MaxRack system is intended to function standalone, 
management nodes are recommended. If you intend to integrate into a larger cluster, management nodes may not 
be necessary. Management nodes need not have the same configuration as other CPU nodes in a MaxRack cluster. 
•	For system management MaxRack head nodes run xcat cluster management software. 
•	For job management, MaxRack head nodes can run open source job distribution systems such as Grid Engine or 
Maxeler can develop a custom engineered solution for your requirements using Maxeler ClusterMap technology.

Rack enclosures must have enough space to support all the nodes, switches and power distribution you have 
selected. Multiple racks may be necessary for large systems.
•	For data center environments, we recommend either 42U or 48U rack enclosures
•	For office environments, we can provide 12U enclosed racks with sound proofing and active noise-cancelling.
It is important to specify power feeds that match the types available in your deployment environment and your 
requirements for the MaxRack system. Power distribution units (PDUs) in the MaxRack will distribute power from 
the input feed to individual nodes.
•	MaxRack systems can be configured with 16A, 32A or 3-phase power feeds. For office environments, small MaxRack 
systems can be connected directly to multiple wall sockets.
•	For power feed redundancy, make sure to only utilize nodes that support redundant power supplies then specify a 
complete set of PDUs for each feed. Each node should be connected to one PDU on feed A and one PDU on feed B, 
and PDUs should only be loaded to half their rated current to allow them to continue functioning if one feed fails.

MPC-X series nodes can be directly connected to CPU nodes via Infiniband cables 
without a switch. Each MPC-X node can have 1 or 2 Infiniband connections which 
can be connected to a single CPU node (to provide maximum bandwidth between 
the MPC-X DFEs and the CPU node) or two CPU nodes (to allow both CPU nodes 
to share a single MPC-X node).  
 
In the illustrated configuration, MPC-X nodes are connected to high density 
CPU-C nodes, providing 32 DFEs and 4 CPU nodes in 6U, with a ratio of 8 DFEs 
to 1 CPU node. Each CPU node can utilize 1 or 2 Infiniband uplinks depending on 
the desired bandwidth.
For disk bound applications, DFE compute resource and disk capability must be 
balanced. If the application requirements are static, MPC-X series nodes can be 
directly connected to high disk capability CPU-B nodes. 
 
In the illustrated configuration, a single MPC-X node is directly connected to two 
CPU-B nodes, providing 8 DFEs and 2 CPU nodes in 5U with a ratio of 4 DFEs to 
1 CPU node. 
For smaller systems, a homogeneous cluster of MPC-C series nodes can be the 
most suitable solution. Individual MPC-C nodes can be configured to handle the 
demands of the application and then this single node unit can be replicated to 
build larger systems.  
 
MPC-C series clusters are also well suited for applications which demand the 
highest levels of bandwidth between DFEs and CPUs, with a direct PCI Express 
connection to each DFE.
Where application requirements are not static at runtime, or multiple 
applications with differing requirements should be able to run on the same 
hardware, MPC-X nodes provide the most flexibility by supporting a switched 
interconnection between multiple MPC-X nodes and multiple CPU nodes. 
Any CPU node can utilize any number of DFEs on the Infiniband network and 
utilization of DFEs is maximized by making them available to any CPU node that 
might wish to use them. Excess CPU node capacity can be utilized to run CPU-
only processes. 
 
In the illustrated configuration, multiple MPC-X series nodes and multiple 
general-purpose CPU-A nodes are connected to a 1U Infiniband switch.

MPC-N series nodes can function as gateways between different networks or 
as a bridge between one or multiple clients and upstream servers. Example 
application areas include firewalls, network forensics and Direct Market Access 
gateways.
Each DFE in an MPC-N series node has two network ports, one of which can be 
connected to the upstream and one to the downstream network.
Distribution arrangements pre-process data from an upstream network and then 
dispatch data to downstream clients. An example of this type of application is 
financial market data transformation, pre-processing exchange data feeds into a 
form optimized for use by downstream nodes. 
 
One network port in an MPC-N DFE can be connected to the upstream network 
and a second can be connected to a downstream Ethernet switch to distribute 
data to multiple downstream clients.
As network endpoints, MPC-N DFEs implement full network clients or servers. 
Example applications include low latency financial trading engines, cache or 
database servers, webservers etc.  
 
A single network port on an MPC-N DFE can be connected to the network, 
or multiple ports can be utilized to either provide additional bandwidth or to 
separate logically different parts of a connection (e.g. a trading engine may 
receive market data on one port, and communicate orders with the exchange on 
the second port).
MPC-N nodes can aggregate multiple upstream data feeds into a single feed, pre-
processing and filtering data before passing a more manageable data stream to 
a downstream node.  
 
One network port in an MPC-N DFE can be connected to an upstream network 
switch and the node can listen to data from many independent data sources 
connected to that switch. A second network port on the DFE can be connected to 
the downstream node.

Maxeler hardware systems contain multiple Dataflow Engines (DFEs) providing the dataflow computing resource. Each DFE 
contains a large configurable logic fabric, off-chip memory and interconnect. DFEs have several kinds of memory resources: 
•	FMEM: ultra fast on-chip memory
•	QMEM: fast off-chip memory with low latency
•	LMEM: large off-chip memory for bulk data storage
DFEs also have interconnect to/from CPU (via Infiniband in MPC-X Series systems, or PCI Express in MPC-C or MPC-N Series) 
and MaxRing connections to other DFEs. Some DFEs also support direct low-latency network connectivity to the DFE. 
Maxeler provides several different DFE architectures optimized for different use cases. Raw specifications are tabulated below.
